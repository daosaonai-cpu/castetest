Crash Course: Decision Making

How your code can think, choose, and react.

Imagine you're following a recipe. Most steps are sequential: "First, chop the onions. Second, heat the oil." But sometimes, a recipe has choices: "If you want it spicy, add chili flakes." This is decision-making. Your code, just like a recipe, needs to react to different situations. Decision-making statements are the tools that allow your program to ask questions and choose a path to follow based on the answers.

1. The if Statement

The if statement is the most basic building block. It's like asking a simple "yes" or "no" question. If the answer is "yes," you perform an action. If the answer is "no," you simply do nothing and move on.

Analogy: The Umbrella Rule ‚òÇÔ∏è

Think about leaving your house: "If it is raining, then I will take an umbrella." If it's not raining (the condition is false), you just ignore that rule and walk out the door.

Syntax

if (condition) {
    // This is the "action block".
    // It only runs if the 'condition' inside the parentheses is true.
}

Example: Accessing a Restricted Movie

#include <stdio.h>

int main() {
    int user_age = 15;
    printf("Checking your age for the movie...\n");

    // The condition 'user_age >= 13' is TRUE because 15 is >= 13.
    if (user_age >= 13) {
        // Because the condition was true, this line will be executed.
        printf("Access granted! Enjoy the movie.\n");
    }

    printf("Thank you for visiting!\n"); // This line runs no matter what.
    return 0;
}

2. The if...else Statement

What if you need a plan for when the condition is false? That's where if...else comes in. It provides an alternative path, a "Plan B." It creates a clear "this or that" choice, guaranteeing that one of the two blocks of code will run.

Analogy: The Ice Cream Plan üç¶

You decide: "If the ice cream truck is here, then I'll buy a cone. Else (otherwise), I'll eat the cookies at home." You are definitely going to have a snack; the if...else structure just determines which snack you have.

Syntax

if (condition) {
    // This block runs if the condition is TRUE.
} else {
    // This block runs if the condition is FALSE.
}

Example: Checking a Password

#include <stdio.h>
#include <string.h> // Needed for comparing strings

int main() {
    char correct_password[] = "secret123";
    char user_input[] = "password";

    // strcmp() returns 0 if the strings are the same.
    // Since "password" is not "secret123", the condition is FALSE.
    if (strcmp(user_input, correct_password) == 0) {
        printf("Login successful! Welcome.\n");
    } else {
        // So, the 'else' block runs instead.
        printf("Incorrect password. Please try again.\n");
    }
    return 0;
}

3. The if...else if...else Ladder

Sometimes one question isn't enough. You might need to check a series of related conditions. The if...else if...else ladder is a clean way to do this. It's like a checklist your program goes through from top to bottom, stopping as soon as it finds a true condition.

Analogy: Grading a Test üìù

A teacher follows a set of rules: "If the score is above 90, the grade is 'A'. Else if the score is above 80, the grade is 'B'. Else if the score is above 70, the grade is 'C'. Else, the grade is 'F'."

Example: The Grading Program

#include <stdio.h>

int main() {
    int score = 85;
    char grade;

    if (score >= 90) {
        grade = 'A';
    } else if (score >= 80) { // 85 >= 80 is TRUE, so this block runs.
        grade = 'B';
    } else if (score >= 70) {
        grade = 'C';
    } else {
        grade = 'F';
    }

    printf("With a score of %d, your grade is %c.\n", score, grade);
    return 0;
}

4. The switch...case Statement

The switch...case statement is a specialized tool for when you have a single variable and you want to perform different actions based on its exact value. It's often cleaner and more efficient than a long if...else if ladder for this specific scenario.

Analogy: A Vending Machine ü•§

A vending machine is like a switch. You provide a single value (the code "B4"). The machine checks your code against its options (cases). If it finds a match (case "B4":), it dispenses a soda. If not (default), it gives an error.

Example: Day of the Week

The break; statement is critical. If you forget it, the program will "fall through" and execute the code in the next case as well!

#include <stdio.h>

int main() {
    int day_of_week = 3; // 1=Sun, 2=Mon, 3=Tue

    switch (day_of_week) {
        case 1: printf("It's Sunday.\n"); break;
        case 2: printf("It's Monday.\n"); break;
        case 3: 
            printf("It's Tuesday.\n"); 
            break; // Finds a match and breaks out.
        case 4: printf("It's Wednesday.\n"); break;
        case 5: printf("It's Thursday.\n"); break;
        case 6: printf("It's Friday.\n"); break;
        case 7: printf("It's Saturday.\n"); break;
        default: printf("Invalid day number.\n");
    }
    return 0;
}

5. The Ternary Operator (? :)

The ternary operator is a compact, inline version of an if...else statement. It's used to assign a value to a variable based on a single condition. Its brevity makes it handy but can reduce readability if overused or nested.

Syntax

variable = (condition) ? value_if_true : value_if_false;

Example: Pass or Fail

#include <stdio.h>
#include <string.h>

int main() {
    int score = 75;
    char* result;

    // If score is >= 60, result is "Pass". Otherwise, it's "Fail".
    result = (score >= 60) ? "Pass" : "Fail";

    printf("With a score of %d, the result is: %s\n", score, result);
    return 0;
}

6. The goto Statement

The goto statement provides an unconditional jump from the goto to a labeled statement in the same function. It tells the program to stop what it's doing and immediately "teleport" execution to another spot.

‚ö†Ô∏è Strong Word of Caution

The use of goto is heavily discouraged in all modern programming. It creates code that is incredibly difficult to read and debug, often called "spaghetti code." For every problem you think goto can solve, there is a better, clearer solution using loops or functions. It is included here for educational completeness only.

Example: (For Educational Purposes Only)

#include <stdio.h>

int main() {
    int count = 1;

start_loop: // This is the label we will jump to.
    printf("Count is: %d\n", count);
    count++;

    if (count <= 3) {
        goto start_loop; // Jump back to the top of our "loop".
    }

    printf("Loop finished.\n");
    return 0;
}

Quick Comparison

Statement Best For Key Feature
if A single, optional action. Executes code only if a condition is true.
if-else A clear "this or that" choice. Guarantees one of two blocks will run.
if-else if Checking multiple, related conditions in order. A chain of checks; stops at the first true one.
switch Checking a variable against many exact values. Clean and often more efficient than long if-ladders.
ternary ?: Assigning one of two values to a variable. A very compact, inline if-else statement.
