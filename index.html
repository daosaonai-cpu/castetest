☁️ AWS AI/ML Deployment

  • Amazon SageMaker: The primary fully-managed service for building, training, and deploying ML models. Use SageMaker Endpoints for secure, real-time HTTP inference.
  • Storage & Containers: Use S3 to store massive datasets and serialized model artifacts (like model.tar.gz). Use ECR (Elastic Container Registry) to store custom Docker images for ML workloads.
  • Compute & Frameworks: Use Deep Learning AMIs (DLAMI) on EC2 to get pre-installed frameworks (PyTorch, TensorFlow) and CUDA drivers. For cost-effective, high-performance inference, AWS offers custom ASICs called AWS Inferentia.
  • Serverless AI: For lightweight models, combining API Gateway + AWS Lambda provides a highly scalable, zero-maintenance architecture.
  • Security: Always use IAM Roles to securely grant services (like SageMaker) permissions to access other resources (like reading from S3) without hardcoding access keys.

🌩️ Google Cloud Platform (GCP)

  • Vertex AI: Google's unified platform bringing all ML services together. It includes Vertex AI AutoML for training high-quality models with minimal ML expertise.
  • Serverless & Containers: Cloud Run is perfect for deploying containerized ML models as serverless HTTP APIs that can automatically scale to zero when not in use.
  • Data & Compute: BigQuery ML allows users to train and evaluate models directly using standard SQL queries. Google's custom hardware accelerators specifically built for machine learning are called TPUs (Tensor Processing Units).
  • Orchestration & Storage: Store built Docker images in Artifact Registry. For highly complex, microservice-based ML apps requiring granular custom orchestration, use GKE (Google Kubernetes Engine).

🌍 Open Source & General Concepts

  • FastAPI: A fast, modern Python web framework highly popular for serving ML models due to asynchronous support and automatic interactive API docs (Swagger UI).
  • Streamlit: An open-source framework that turns Python data scripts into interactive, shareable web apps in minutes—without needing to know HTML/CSS.
  • ML Lifecycle: MLflow is a widely used open-source platform for experiment tracking, reproducibility, and managing a central model registry.
  • Model Interoperability: ONNX (Open Neural Network Exchange) allows models to be trained in one framework (e.g., PyTorch) and easily deployed or optimized in another environment.
  • Hugging Face: A collaborative open-source hub for hosting NLP models, datasets, and easily deploying machine learning demos via "Spaces".
  • Concepts to Know: A/B Testing involves routing live traffic to old (A) and new (B) models in production to compare performance. Avoid unpickling models from untrusted sources using Python's pickle module, as unpickling can execute arbitrary malicious code on your machine.

🐳 Docker & Containerization Basics

  • Core Concept: Docker packages applications and their dependencies into standardized units for deployment. An Image is the static, read-only blueprint; a Container is the live, running instance of an image.
  • Dockerfile: The text document containing all the sequential instructions a user could call on the command line to assemble a Docker image.
  • Essential Commands:
    • docker build -t my-app . : Builds an image from a Dockerfile located in the current directory (.) and tags it with a name.
    • docker run -p 8000:80 my-image : Runs a container and maps port 8000 on your host machine to port 80 inside the container.
    • docker ps : Lists all currently active/running containers on your system.
  • Storage & Orchestration: Use Docker Volumes to persist data generated by the container so it isn't lost when the container is destroyed. Use Docker Compose (via a docker-compose.yml file) to define and spin up multi-container applications easily.
