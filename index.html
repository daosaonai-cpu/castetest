C Programming Concept Guide

1. File Pointers & Writing to Files

What is a File Pointer?

A File Pointer is a pointer to a structure of type FILE. It points to information that defines various things about the file, including its name, status, and the current position of the stream. It acts as a handle that the program uses to keep track of the file being accessed.

C Program to Write to a File

#include <stdio.h>
#include <stdlib.h>

int main() {
    FILE *fptr;

    // Open file in write mode ("w")
    // If file doesn't exist, it creates it.
    fptr = fopen("example.txt", "w");

    if (fptr == NULL) {
        printf("Error opening file!\n");
        exit(1);
    }

    // Write text to the file
    fprintf(fptr, "Hello, this is a text written by a C program.\n");
    fprintf(fptr, "File handling is useful.");

    // Close the file to save data
    fclose(fptr);

    printf("Data written successfully.");
    return 0;
}

2. Functions: Advantages, Calls, and Definitions

Advantages of Using Functions

  • Reusability: Write code once and use it multiple times.
  • Modularity: Breaks down complex problems into smaller, manageable chunks.
  • Maintainability: Debugging is easier because errors are localized to specific functions.
  • Abstraction: Hides implementation details from the user (they only need to know what the function does, not how).

Function Definition vs. Function Call

Concept Explanation
Function Definition This is the actual body of the function. It contains the code block that executes the task. It specifies the return type, name, parameters, and logic.
Example: int add(int a, int b) { return a + b; }
Function Call This is the instruction that tells the compiler to execute the function at that specific point. It transfers control to the function definition.
Example: int sum = add(5, 10);

3. Function Prototypes

What is a Function Prototype?

A function prototype is a declaration of a function that tells the compiler about the function's name, return type, and parameters before the function is actually defined.

Syntax: return_type function_name(type arg1, type arg2, ...);

Is it Mandatory?

No, it is not strictly mandatory, but it is highly recommended.

Justification:

  • If you define your function before the main() function (or before it is called), a prototype is not needed because the compiler already knows about it.
  • However, if you define the function after it is called, the compiler will raise a warning or error (implicit declaration) without a prototype. Prototypes ensure strict type checking on arguments.

4. Pointers in C

What is a Pointer Variable?

A pointer variable is a variable that stores the memory address of another variable rather than a value.

Advantages of Using Pointers

  • Dynamic Memory Allocation: Allows creating data structures (like linked lists, trees) that can grow or shrink during runtime using malloc and calloc.
  • Efficient Handling of Arrays/Structures: Passing a pointer to a large structure or array is more efficient than passing the actual data by value (which requires copying).
  • Hardware Access: Pointers allow direct access to memory locations, useful for systems programming.
  • Return Multiple Values: Pointers allow functions to modify variables in the calling scope, effectively "returning" multiple results.

Relationship Between Arrays and Pointers

There is a very close relationship between arrays and pointers in C:

  • The name of an array acts as a constant pointer to the first element of the array.
  • If you have an array int arr[5], then arr is equivalent to &arr[0].
  • Array arithmetic and pointer arithmetic are equivalent: accessing arr[i] is internally converted by the compiler to *(arr + i).

5. Program: Average of Array Elements

#include <stdio.h>

// Function Definition
float calculateAverage(int arr[], int n) {
    int sum = 0;
    for (int i = 0; i < n; i++) {
        sum += arr[i];
    }
    return (float)sum / n; // Typecast to float for decimal precision
}

int main() {
    int n;

    printf("Enter number of elements: ");
    scanf("%d", &n);

    int numbers[n];
    printf("Enter %d integers:\n", n);
    for(int i = 0; i < n; i++) {
        scanf("%d", &numbers[i]);
    }

    // Function Call
    float avg = calculateAverage(numbers, n);

    printf("The average is: %.2f\n", avg);

    return 0;
}

6. Macros vs. Functions

What is a Macro?

A macro is a fragment of code that has been given a name. Whenever the name is used, it is replaced by the contents of the macro before compilation (during the pre-processing stage). Defined using #define.

Difference Between Macro and Function

Feature Macro Function
Processing Processed by the Preprocessor (text substitution). Processed by the Compiler and executed at runtime.
Speed Faster execution (no overhead of context switching). Slower than macros due to overhead of call/return.
Code Size Increases code size (code is expanded at every usage). Keeps code size smaller (code exists in one place).
Type Checking No type checking. Strict type checking is performed.

7. Understanding 'void'

What is void?

The keyword void in C means "nothing" or "no type." It is an empty data type.

When is void used in a function?

  1. As a Return Type: When a function does not return a value to the calling function.
    Example: void printHello() { printf("Hello"); }
  2. As an Argument List: When a function takes no parameters.
    Example: int getRandomNumber(void);
