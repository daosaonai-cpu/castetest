Key Concepts to Remember:

1. Network Topologies (The Layout): This describes the physical or logical arrangement of devices (nodes) and connections on a network.
  • Star: All nodes connect to a single, central device (like a switch or hub). Pro: It's easy to manage, add/remove devices, and a single cable failure only affects one device. Con: It has a single point of failure; if the central switch fails, the entire network segment goes down.
  • Bus: A legacy topology where all nodes share a single main cable (the "backbone"). Pro: Simple and cheap to install. Con: It's hard to troubleshoot, a break in the main cable takes down the whole network, and performance degrades heavily with many devices due to data collisions.
  • Ring: Devices are connected in a continuous loop. Data packets travel from node to node, often in one direction using a "token" (Token Ring) to manage access and prevent collisions. Con: A failure in one node or cable can break the entire ring.
  • Mesh: A highly reliable topology where nodes are interconnected. In a Full Mesh, every node connects to every other node, providing maximum redundancy. In a Partial Mesh, only key nodes are fully interconnected. Pro: Very fault-tolerant, as data can be rerouted. Con: Extremely expensive and complex to install and manage.
  • Hybrid: A combination of two or more topologies. A common example is a "Star-Bus" network, where several Star topology workgroups are connected via a linear Bus backbone.
2. IP Addressing (The Address System): This system provides a unique numerical identifier for every device on a network, allowing them to find and communicate with each other.
  • IPv4: The 32-bit address format (e.g., 192.168.1.100). It's written as four 8-bit numbers (octets) ranging from 0 to 255. The loopback address (for testing) is 127.0.0.1.
  • IPv6: The 128-bit address format, written in hexadecimal (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334). It supports a vastly larger number of addresses. The loopback address is ::1. Addresses starting with fe80:: are Link-Local, meaning they are only used for communication on the same local network link.
  • Subnet Mask: A 32-bit number (like 255.255.255.0) that divides an IPv4 address into two parts: the Network ID (which network it's on) and the Host ID (which device it is). The network ID is constant for all devices on the same subnet.
  • CIDR Notation: A modern shorthand for the subnet mask. It uses a slash followed by the number of bits in the Network ID. For example, 192.168.1.0/24 means the first 24 bits are for the network, which is the same as the subnet mask 255.255.255.0.
  • Private IP Ranges: Addresses reserved for internal networks (not reachable on the public internet). The main ranges are: Class A (10.0.0.0 - 10.255.255.255), Class B (172.16.0.0 - 172.31.255.255), and Class C (192.168.0.0 - 192.168.255.255).
  • Special Addresses: Within a subnet, the first address is the Network ID (e.g., 192.168.10.0) and the very last is the Broadcast Address (e.g., 192.168.10.255), which sends data to all devices on that subnet.
  • Key Protocols: DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses, subnet masks, and default gateways to devices when they join a network. DNS (Domain Name System) is the internet's "phonebook," translating human-friendly domain names (like google.com) into their corresponding IP addresses.
  • OSI Model Layers: Layer 2 (Data Link) uses MAC addresses (physical hardware addresses) for local network communication. Layer 3 (Network) uses IP addresses to route data across different networks.
