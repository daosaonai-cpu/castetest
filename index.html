C Programming I/O

A Crash Course on Managing Input & Output Operators

What is Input/Output in C?

In programming, getting data into your program and getting results out are fundamental operations. These are known as Input and Output (I/O). In C, we handle I/O using a set of functions defined in the standard I/O library, which you include in your code with #include <stdio.h>.

  • Input: Feeding data to your program. This can be from a keyboard, a file, or another device. The standard input stream is called stdin.
  • Output: Displaying data from your program. This can be to the screen, a file, or a printer. The standard output stream is called stdout.

1. Reading and Writing a Character

C provides simple functions specifically for handling single characters. These are great for simple tasks like reading a single menu choice or processing text one character at a time.

Reading a Character: getchar()

The getchar() function reads the next available character from the standard input (stdin), which is usually the keyboard. It returns the character read as an integer. 


#include <stdio.h>

int main() {
    int ch; // Use int to hold the return value of getchar()

    printf("Please enter a character: ");
    
    // Read a single character from the keyboard
    ch = getchar(); 

    printf("You entered the character: %c\n", ch);

    return 0;
}

Writing a Character: putchar()

The putchar() function writes a single character to the standard output (stdout), which is usually the screen. 


#include <stdio.h>

int main() {
    char myChar = 'A';

    printf("The character is: ");

    // Write a single character to the screen
    putchar(myChar);
    
    // putchar can also print a newline
    putchar('\n'); 

    return 0;
}

2. Formatted Output: printf()

While putchar() is good for single characters, you'll usually want to print more complex things like numbers, strings, and formatted text. For this, C gives us the powerful printf() function.

The name printf stands for "print formatted". It allows you to construct a string with placeholders, called format specifiers, which are replaced by the values of variables you provide.

Syntax

printf("format string", argument1, argument2, ...);

Common Format Specifiers

Specifier Data Type Example
%d or %i Signed Integer 123, -45
%f Floating-point number (float, double) 3.14159
%c Single Character 'a'
%s String (array of characters) "Hello World"
%% Literal '%' sign Prints a single %

Example: Using printf()


#include <stdio.h>

int main() {
    int age = 25;
    float pi = 3.14159;
    char grade = 'A';
    char name[] = "Alex";

    // Printing different data types using their format specifiers
    printf("Hello, my name is %s.\n", name);
    printf("I am %d years old.\n", age);
    printf("My favorite letter is %c, and my grade was %c.\n", grade, grade);
    printf("The value of Pi is approximately %f.\n", pi);

    // You can also format floats to a specific precision
    printf("Pi to two decimal places is %.2f.\n", pi);
    
    printf("I am 100%% sure about this.\n");

    return 0;
}

3. Formatted Input: scanf()

Just as printf() handles formatted output, scanf() handles formatted input. It reads data from the standard input (keyboard) according to the format specifiers you provide and stores the data in variables.

Syntax

scanf("format string", &variable1, &variable2, ...);

The Ampersand & is Crucial!

Notice the & symbol before each variable name. This is the "address-of" operator. It gives scanf() the memory address of the variable, so it knows where to store the input data. Forgetting the & is one of the most common C programming mistakes and will cause your program to crash! (The only common exception is when reading into a string/character array, as the array name itself points to the first element's address).

Example: Using scanf()


#include <stdio.h>

int main() {
    int userAge;
    float userHeight;
    char userInitial;
    char userName[20]; // A character array to store a string

    // Prompt the user for their name and read it
    printf("Enter your first name: ");
    scanf("%s", userName); // No '&' needed for character arrays

    // Prompt the user for their age and read it
    printf("Enter your age: ");
    scanf("%d", &userAge); // Use '&' to get the address

    // Prompt for height
    printf("Enter your height in meters (e.g., 1.75): ");
    scanf("%f", &userHeight);

    // Prompt for initial
    printf("Enter your middle initial: ");
    scanf(" %c", &userInitial); // Note the space before %c

    // Display the collected information
    printf("\n--- User Information ---\n");
    printf("Name: %s\n", userName);
    printf("Age: %d\n", userAge);
    printf("Height: %.2f meters\n", userHeight);
    printf("Middle Initial: %c\n", userInitial);

    return 0;
}

Pro Tip: The Space Before %c

In the example above, we used " %c" (with a leading space) in the scanf for the character. Why? When you type input like "1.75" and press Enter, the newline character (\n) from the Enter key is left in the input buffer. The next scanf("%c", ...) would read that newline character instead of waiting for your input. The leading space in " %c" tells scanf to skip any whitespace (including newlines) before reading the actual character.
