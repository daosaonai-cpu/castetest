What is a Function?

In C, a function is a self-contained block of code that performs a specific, single, well-defined task.

Think of a function as a **specialist** you can hire. Instead of your main() function trying to do every single job by itself (calculating, printing, getting user input, etc.), it can "delegate" tasks to these specialists.

For example, you can create an addNumbers() specialist. Its only job is to add two numbers. You "call" this specialist, give it the numbers (called arguments), it performs the calculation, and then it "returns" the result to you.

This has two huge benefits:

  • Reusability: Instead of writing the 10 lines of code for a complex task every time you need it, you write it *once* inside a function. Then, you can "call" that function 100 times from different places in your program. This saves time and reduces errors.
  • Organization: It keeps your code clean and readable. Your main() function becomes a simple "to-do" list, calling other functions to get the work done. This makes it much easier to understand, debug, and maintain your program.

1. The Form (Syntax) of a C Function

All C functions follow a specific blueprint, often called the "function definition." It tells the compiler everything it needs to know about the function. 

return_type function_name(parameter_list) {
    // Body of the function
    type local_variable; // Variables declared here only exist here
    // ... statements that do the work ...
    
    return value; // Send a value back
}

Let's break that down in detail:

  • return_type: This is the data type of the value the function will send back *after* it's done.
    • It can be any valid C type: int, float, double, char, etc.
    • If the function's job is just to *do* something (like print to the screen) and not send a value back, you must use the special keyword void. This means "returns nothing."
  • function_name: The unique name you give your function.
    • It follows the same rules as variable names (must start with a letter or underscore, can contain letters, numbers, underscores).
    • Good names are descriptive and often use verbs, like calculateArea, printReport, or getUserInput.
  • (parameter_list): This is the function's "input." It's a list of variables that the function needs to receive to do its job.
    • Each parameter must have a type and a name, like (int num1, int num2) or (float radius, char unit).
    • These parameters act as local variables inside the function.
    • If the function takes no inputs, you should write (void), like int getRandomNumber(void). Many modern compilers also allow empty parentheses ().
  • { ... } (The Body): The curly braces and all the code inside them are the "body" of the function.
    • This is the code that actually runs when the function is called.
    • Any variables you declare inside the body are **local** to that function. They are created when the function starts and destroyed when it ends. They cannot be seen or used by any other function (including main()).

2. How to "Call" a Function

Defining a function is like writing a recipe. It doesn't do anything until you actually *use* it. Using a function is called "calling" it.

When you call a function, you pass it values for its parameters. These values are called arguments.

Parameters vs. Arguments:

  • Parameters are the variables in the function's *definition* (the "inputs" it expects). E.g., int a, int b.
  • Arguments are the actual values you *pass* to the function when you *call* it. E.g., 5, 10.

Calling a Function That Returns a Value

If a function has a return type (like int), it will send a value back. You almost always want to "catch" this value in a variable. 

int addNumbers(int a, int b) {
    return a + b;
}

int main(void) {
    int sum1 = addNumbers(5, 3);  // Call the function, store the result in sum1
    int sum2 = addNumbers(100, 200); // Call it again with different arguments

    printf("The first sum is: %d\n", sum1); // Prints 8
    printf("The second sum is: %d\n", sum2); // Prints 300
    
    return 0;
}

Calling a Function That Returns void

If a function returns void, it doesn't send a value back. You just call it on its own line. You cannot try to store its "result" in a variable. 

void greetUser(char* name) {
    printf("Hello, %s! Welcome.\n", name);
}

int main(void) {
    greetUser("Alice"); // Just call the function. It does its job (prints)
    greetUser("Bob");   // Call it again.

    // int x = greetUser("Charlie"); // ERROR! Can't assign 'void' to 'int'
    
    return 0;
}

3. Function Prototypes (Declarations)

The C compiler reads your code from top to bottom. If you try to *call* a function before you have *defined* it (or before the compiler has read its definition), the compiler will give you an error because it doesn't know that function exists.

There are two ways to solve this:

  1. Define all your functions *before* main(). This works, but can be messy.
  2. Use Function Prototypes. This is the preferred way.

A prototype (or declaration) is a single line, usually at the top of your file (before main()), that tells the compiler a function's "signature" *without* giving the body. It's like saying, "Hey compiler, I promise that *later* I will define a function with this name, these parameters, and this return type."

A prototype is just the function's first line, ending with a semicolon ;. 

// main.c
#include <stdio.h>

// --- Function Prototypes ---
void greetUser(char* name);
int addNumbers(int a, int b);

// --- Main Function ---
int main(void) {
    // This is OK! The compiler saw the prototypes.
    greetUser("Alice");
    int sum = addNumbers(10, 20);
    printf("Sum is: %d\n", sum);
    return 0;
}

// --- Function Definitions ---
// Now we provide the full "recipe" for each function
void greetUser(char* name) {
    printf("Hello, %s! Welcome.\n", name);
}

int addNumbers(int a, int b) {
    return a + b;
}

4. Return Values and Types

The return_type is a strict promise. If you tell the compiler your function will return an int, it *must* return a valid int.

Example: Returning a Number (int)

This function promises to return an int. The return sum; line fulfills that promise. 

int addNumbers(int a, int b) {
    int sum = a + b;
    return sum; // We send back the 'sum', which is an int
}

Example: Returning a Decimal (float)

Here, the function promises a float. Note that we use 3.14159f and 2.0f to make sure the numbers are treated as floats, not doubles. 

float calculateCircleArea(float radius) {
    float pi = 3.14159f;
    float area = pi * radius * radius;
    return area; // Fulfills the 'float' promise
}

Example: Returning Nothing (void)

This function promises void (nothing). Its job is purely to perform an *action* (printing). It does not need a return statement at the end, as it will automatically return when it hits the closing brace }. 

void greetUser(char* name) {
    printf("Hello, %s! Welcome.\n", name);
    // No 'return' statement is needed here.
}

5. The return Statement

The return keyword is a powerful command. It does two critical things simultaneously:

  1. It immediately exits the function. As soon as the program hits a return, the function stops running. Any code *after* the return statement (in the same block) will be ignored. This is called "unreachable code."
  2. It sends a value back (unless the function is void) to the exact spot where the function was called. The program execution then continues from that point.

The value you return must match the return_type you promised, or be safely convertible to it. 

int giveMeFive() {
    return 5; // This is valid. We promised an int, we returned an int.
    printf("Hello!"); // UNREACHABLE CODE. This will never run.
}

int wrongFunction() {
    return "hello"; // COMPILE ERROR! We promised an int,
                       // but we are trying to return a string literal (char*).
}

Using return in a void Function

You *can* use return; (with no value) in a void function. This is commonly used to "exit early," often for validation or error-checking. 

void printPositiveNumber(int num) {
    if (num < 0) {
        printf("Error: Number is negative.\n");
        return; // Exit the function immediately.
    }
    
    // This line will only run if the 'if' condition was false.
    printf("Your positive number is: %d\n", num);
}

Key Takeaways

  • Functions are reusable and organizing blocks of code.
  • The function **definition** is the full recipe: return_type name(parameters) { ... }.
  • A **prototype** is a declaration (like int add(int, int);) that tells the compiler a function exists before it's defined.
  • Parameters are the variables in the definition; Arguments are the values you pass when you *call* the function.
  • The return_type is a strict promise of what data type the function will send back.
  • void is a special type that means "I promise to return nothing."
  • The return keyword immediately stops the function and (if not void) sends one value back.
